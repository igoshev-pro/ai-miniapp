# Проект: Миниап генерации аудио, видео, изображений и текста с AI-провайдерами

## Общее описание

Данный проект представляет собой фронтенд на Next.js 14+ для платформы Telegram WebApp, реализующий генерацию контента с помощью AI-провайдеров. Среди поддерживаемых возможностей — генерация текста, изображений, видео и аудио с использованием современных моделей OpenAI, ElevenLabs, Suno и других.

Ключевая особенность — интеграция с backend, предоставляющим API для различных типов генерации, с поддержкой состояния пользователя (баланс токенов), асинхронных задач, WebSocket для обновления статусов и избранное.

---

## Архитектура

- **Frontend**: React (Next.js), с использованием хуков (`useGeneration`, `useUser`, `useFavorites` и др.), контекста Telegram и UI-компонентов (`MediaResult`).
- **Backend**: REST API с асинхронной генерацией, поддержкой разных провайдеров (KIE.ai для ElevenLabs и Suno), проверкой баланса, очередями задач и WebSocket-уведомлениями.
- **WebSocket**: Используется для передачи статусов генерации (прогресс, ошибки, завершение) в реальном времени.
- **State management**: Используется Zustand для управления состоянием генераций, избранного, пользователей и других сущностей.

---

## Компоненты

- `AudioGenerationPage.tsx` — страница для генерации аудио с поддержкой моделей ElevenLabs и Suno, расширенными параметрами и загрузкой аудио для распознавания.
- `MediaResult.tsx` — компонент отображения результата генерации с плеером (аудио/видео/изображение), кнопками скачивания, избранного и повторного запроса.
- Хуки: 
  - `useGeneration` — для запроса генерации, подписки на статусы, управления историей и избранным.
  - `useFavorites` — управление состоянием избранного с оптимистичным обновлением и синхронизацией с API.
  - `useUser` — данные о пользователе и балансе.

## Текущая функциональность аудио

- Поддержка генерации аудио через две группы моделей:
  - **ElevenLabs**: текст→речь (TTS), речь→текст (STT), изоляция аудио (удаление шума/голосов), создание звуковых эффектов, синтез диалогов.
  - **Suno**: генерация музыки и песен с расширенными параметрами (customMode, instrumental, style, duration и пр.).
- Передача расширенных параметров в запросах генерации.
- Загрузка аудиофайлов для моделей, работающих с файлами (speech-to-text, audio-isolation).
- Подписка на статус генерации по WebSocket с отображением прогресса и уведомлениями.
- Возможность скачивания аудио файлов через UI, учитывая ограничения Telegram WebApp.
- Добавление генераций в избранное, хранение состояния и синхронизация с сервером.

## Контекст фичи: Расширение аудиогенерации

### Задачи

- Полная поддержка всех моделей ElevenLabs с учётом их специфичных параметров и типов генерации (TTS, STT, изоляция, эффекты).
- Поддержка всех ключевых моделей Suno с дополнительными настройками и параметрами.
- UI: переключатель моделей с отображением релевантных настроек для каждой.
- Обеспечение возможности загрузки аудиофайлов для моделей ElevenLabs, которые требуют входящий аудиофайл.
- Генерация запросов в формате, соответствующем спецификации backend.
- Интерактивное отображение прогресса генерации с использованием WebSocket, оповещения пользователя.
- Полная поддержка скачивания сгенерированных аудиофайлов.
- Встроенный функционал работы с избранным (добавление/удаление из избранного).

### Проблемы и решения

- **Типизация**: корректное использование типов для параметров и атрибутов React компонентов.
- **UI/UX**: динамическая отрисовка дополнительных настроек по выбранной модели.
- **Загрузка файлов**: асинхронная загрузка аудио с обработкой ошибок.
- **WebSocket**: подписка и отмена подписки для корректного обновления статуса генерации.
- **Оптимизация**: оптимистичное обновление состояния избранного, разделение логики по хук.
